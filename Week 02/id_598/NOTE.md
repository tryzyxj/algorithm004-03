# NOTE

  

## 递归的三大思维

1. 不要人肉进行递归（最大的误区）
2. 找到最近最科方法，将其拆解成可重复解决的问题（重复子问题）
3. 数学归纳法思维

## 递归函数的组成(代码模板)

只要是递归函数，就按照以下的模板去写：

1. 终止条件
2. 处理当前层逻辑
3. 处理下一层
4. 清理当前层

## 算法面试四件套

1. clarification 和面试官过题目
2. possible solutions - > optimal (time & space) 从可选的解决方案中找最优解：时间和空间复杂度最好
3. code，写代码
4. test cases， 测试样例
